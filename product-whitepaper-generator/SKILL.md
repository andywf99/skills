# 产品白皮书生成器 (Product Whitepaper Generator)

根据项目代码结构和业务逻辑，自动生成完整的产品白皮书文档。

## 执行流程

> **核心原则**：采用 **分阶段串行执行**，每扫描完一个模块立即填充文档对应章节，避免一次性加载所有信息导致上下文过长。

### 执行流程总览

```
阶段1: 项目结构扫描 → 生成文档骨架
         ↓
阶段2: 逐模块扫描 → 立即填充对应章节
         ├── Controller 层 → 填充 Controller 章节
         ├── Service 层 → 填充 Service 章节
         ├── Mapper 层 → 填充 Mapper 章节
         ├── Entity 实体类 → 填充 Entity 章节
         ├── Feign 客户端 → 填充 Feign 章节
         ├── Cache/Task/AOP → 填充对应章节
         ├── 消息中间件 → 填充消息章节
         └── Elasticsearch → 填充 ES 搜索引擎章节
         ↓
阶段3: 收尾 → 更新时间戳、验证完整性
```

### 阶段1：项目结构扫描 → 生成文档骨架

**目标**：确认模板和输出路径，生成空文档骨架。

**操作步骤**：

1. **确认模板和输出路径**
   - 询问用户是否有现成的白皮书模板文件
   - 确认输出文件路径（默认为项目根目录下的 `{项目名}产品白皮书.md`）

2. **扫描项目基础结构**
   - 读取 `pom.xml` 获取项目名称、版本、技术栈
   - 扫描 `src/main/java` 目录结构
   - 识别项目架构模式（单模块/多模块、微服务/单体应用）

3. **生成文档骨架**
   ```markdown
   # 【项目名】产品白皮书 / 功能全景图
   ```

### 阶段2：逐模块扫描 → 立即填充对应章节

**核心原则**：扫描一个模块后立即写入文档，保持上下文轻量。

**扫描目标**：`src/main/java/**/controller/`

- 接口路径和功能

**填充章节**：

## 文档模板

完整文档结构模板已提取到 `product-whitepaper-template.md` 文件中。

使用模板时，执行以下步骤：

1. **读取模板文件**：从 `product-whitepaper-template.md` 读取模板内容
2. **替换占位符**：将模板中的占位符替换为实际内容
   - 【项目名】 → 实际项目名称
   - YYYY-MM-DD HH:mm:ss → 当前时间
   - X.X.X → 项目版本
   - main/develop/xxx → 当前分支名
3. **填充章节**：根据扫描结果填充各章节内容
4. **输出文档**：保存为 `{项目名}产品白皮书.md`

### 执行注意事项
1. **分阶段执行**：严格遵循阶段1→阶段2→阶段3的顺序，每完成一个阶段立即输出进度
2. **串行扫描**：同一阶段内的模块扫描串行执行，避免上下文过长
3. **即时写入**：每扫描完一个模块立即更新文档，不要等所有扫描完成再写入
4. **进度反馈**：每完成一个模块扫描，输出进度提示（如"Controller层扫描完成，已填充5.核心功能详解章节"）
