# 产品白皮书生成器 (Product Whitepaper Generator)

根据项目代码结构和业务逻辑，自动生成完整的产品白皮书文档。

## 执行步骤

### 第一步：确认模板和输出路径
1. 询问用户是否有现成的白皮书模板文件
2. 确认输出文件路径（默认为项目根目录下的 `{项目名}产品白皮书.md`）

### 第二步：代码结构分析
使用 Explore agent 或手动分析以下内容：

**必须分析的目录和文件**：
```
src/main/java/
├── controller/     # 接口层：API端点、业务入口
├── service/        # 业务层：核心业务逻辑
├── entity/         # 实体层：数据模型、表结构
├── enums/          # 枚举：状态、类型定义
├── dto/            # 数据传输对象
├── vo/             # 视图对象
├── mapper/         # 数据访问层
├── config/         # 配置类
├── feign/          # 外部服务调用
└── stream/         # 消息监听
```

**关键分析点**：

1. **状态枚举**：查找 `*StatusEnum.java`、`*StateEnum.java`
   - 提取所有状态码和状态名称
   - 分析状态流转逻辑

2. **类型枚举**：查找 `*TypeEnum.java`、`*CodeEnum.java`
   - 提取业务类型分类
   - 分析类型层级关系

3. **处理类型**：查找 `*ProcessTypeEnum.java`、`*OperationEnum.java`
   - 提取操作类型列表

4. **核心实体**：分析主表实体类
   - 字段含义和业务用途
   - 实体间关联关系

5. **Service接口**：分析业务方法
   - 核心业务能力
   - 流程节点操作

6. **Controller**：分析API端点
   - 接口路径和功能
   - 请求参数结构

7. **消息监听**：分析 `@StreamListener` 方法
   - 消息来源渠道
   - 自动化处理逻辑

1. 检查文档结构完整性
2. 确保状态流转图逻辑正确
3. 核对ER模型图关联关系
4. 补充遗漏的业务细节

## 输出格式
- Markdown格式
- 中文撰写
- 表格使用GFM语法
- 流程图使用ASCII字符
- ER图使用ASCII字符

## 关键提示词

### 状态流转分析提示词
```
分析以下状态枚举和Service方法，绘制完整的状态流转图：
1. 列出所有状态码及其含义
2. 分析每个操作方法会导致的状态变更
3. 用ASCII流程图展示状态流转路径
```

### ER模型分析提示词
```
分析以下实体类，生成ER模型图：
1. 识别主表和从表
2. 分析实体间的关联关系（1:1, 1:N, N:N）
3. 提取外键关联字段
4. 用ASCII ER图展示表关系
```

### 功能模块分析提示词
```
分析Controller和Service代码，提取功能模块：
1. 按API路径分组识别功能模块
2. 分析每个模块的业务流程节点
3. 提取每个节点的输入参数、状态变更、输出结果
```

## 注意事项

### 执行注意事项
1. **分阶段执行**：严格遵循阶段1→阶段2→阶段3的顺序，每完成一个阶段立即输出进度
2. **串行扫描**：同一阶段内的模块扫描串行执行，避免上下文过长
3. **即时写入**：每扫描完一个模块立即更新文档，不要等所有扫描完成再写入
4. **进度反馈**：每完成一个模块扫描，输出进度提示（如"Controller层扫描完成，已填充5.核心功能详解章节"）

### 内容质量注意事项
1. 充分挖掘代码中的业务细节，不要遗漏
2. 状态流转图要包含所有分支路径，标注流转条件
3. ER图要清晰展示主从关系和关联类型
4. 核心功能按流程节点逐个细化，包含参数说明
5. 技术栈版本从pom.xml获取，版本号要准确

### 输出格式注意事项
1. Markdown格式，中文撰写
2. 表格使用GFM语法
3. 流程图使用ASCII字符，保持可读性
4. ER图使用ASCII字符，清晰展示关系
5. 文档末尾标注版本和生成日期

### 验证检查清单
- [ ] 所有章节都有内容（或标注待补充）
- [ ] 状态流转图逻辑正确，没有死环
- [ ] ER模型图关联关系完整
- [ ] 技术栈版本号准确
- [ ] 接口清单完整
- [ ] 核心功能详解覆盖所有Controller
